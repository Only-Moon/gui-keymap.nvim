1. gui-keymap.nvim                                               *gui-keymap.nvim*

`gui-keymap.nvim` provides GUI-style shortcuts for users moving from editors
like VSCode, Sublime, and JetBrains IDEs.

It includes safe keymap registration, optional adaptive hints, optional Yanky
status integration, and diagnostics.

==============================================================================
2. Configuration                                            *gui-keymap-config*

>
    require("gui-keymap").setup({
      undo_redo = true,
      clipboard = true,
      select_all = true,
      delete_selection = true,
      shift_selection = true,
      word_delete = true,
      yanky_integration = true,
      hint_enabled = true,
      hint_repeat = 3,
      which_key_integration = true,
      enforce_on_startup = true,
      force_priority = true,
      show_welcome = false,
      preserve_mode = true,
    })
<

Use `hint_repeat = -1` to always show hints.

Legacy per-shortcut toggles are also accepted:
`copy`, `paste`, `cut`, `undo`, `redo`.

3. Keymaps                                                    *gui-keymap-keymaps*

UNDO/REDO ~
- `<C-z>` (normal) -> `u`
- `<C-y>` (normal) -> `<C-r>`
- `<C-z>` (insert) -> `<C-o>u`
- `<C-y>` (insert) -> `<C-o><C-r>`

CLIPBOARD ~
- `<C-c>` (visual) -> `"+y`
- `<C-c>` (normal) -> `"+yy`
- `<C-x>` (visual) -> `"+d`
- `<C-v>` (normal/insert) -> `"+p`

SELECT ALL ~
- `<C-a>` (normal) -> `ggVG`
- `<C-a>` (insert) -> `<Esc>ggVG`

DELETE SELECTION ~
- `<BS>` (visual) -> `"_d`
- `<Del>` (visual) -> `"_d`

SHIFT SELECTION ~
- Normal: `<S-Left>`, `<S-Right>`, `<S-Up>`, `<S-Down>`
- Visual: `<S-Left>`, `<S-Right>`, `<S-Up>`, `<S-Down>`
- Insert: `<S-Left>`, `<S-Right>`, `<S-Up>`, `<S-Down>`

WORD DELETE ~
- `<C-BS>` (normal) -> `db`
- `<C-Del>` (normal) -> `dw`
- `<C-BS>` (insert) -> `<C-o>db`
- `<C-Del>` (insert) -> `<C-o>dw`

==============================================================================
4. Commands                                                 *gui-keymap-commands*

- `:GuiKeymapInfo` - show active maps, conflicts, integration state
- `:GuiKeymapHintReset` - reset hint counters
- `:GuiKeymapRefresh` - re-apply gui-keymap mappings
- `:GuiKeymapDemo` - open a safe demo scratch buffer
- `:GuiKeymapShowcase` - open an expanded demo scratch buffer
- `:checkhealth gui-keymap` - run plugin health checks

==============================================================================
5. Demo                                                           *gui-keymap-demo*

Use `:GuiKeymapDemo` to open a temporary scratch buffer and test shortcuts
without affecting your files.
